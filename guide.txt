Project Title:
Hybrid Feature Fusion Framework for Just-in-Time Software Defect Prediction

Research Objective:
Build a modular machine learning framework for software defect prediction using three feature sources:

1. Traditional JIT defect prediction metrics
2. CodeBERT embeddings
3. LLM-generated reasoning features

Dataset Characteristics:

* More than 100,000 commits
* Multiple programming languages (Java, Python, C++, etc.)
* Binary classification:
  0 = Non-buggy
  1 = Buggy
* Imbalanced class distribution
* Imbalanced language distribution
* Commit timestamps available

Initial Development Dataset:

* Filter commits from years 2018-2026
* Perform stratified sampling based on:
  Language × Bug Label
* Create approximately 10,000 commit subset
* Preserve temporal ordering

Temporal Evaluation Strategy:

* Sort commits chronologically
* First 80% → Train
* Last 20% → Test
* No random splitting
* No data shuffling

Feature Sources:

Traditional JIT Features:

* LA
* LD
* NF
* NS
* ND
* Entropy
* NDEV
* AGE
* NUC
* AEXP
* AREXP
* ASEXP

CodeBERT Features:

* Precomputed CodeBERT embeddings
* Original dimension: 768
* PCA already applied offline
* Final embedding dimension: 384
* Embeddings merged using commit_id
* No PCA required during experiments

LLM Reasoning Features:

Reasoning features are generated separately using an LLM.

For each commit, the LLM produces structured reasoning attributes:

1. Intent
   - bug_fix
   - feature_addition
   - feature_modification
   - refactoring
   - performance_improvement
   - documentation
   - configuration_change
   - test_update
   - dependency_update

2. Change Type
   - control_flow
   - data_manipulation
   - api_change
   - exception_handling
   - ui_change
   - database_change
   - configuration_change
   - test_change
   - security_change

3. Risk Level
   - low
   - medium
   - high
   - critical

4. Complexity
   - low
   - medium
   - high

5. Scope
   - single_line
   - single_file
   - multiple_files
   - cross_module

6. Test Impact
   - yes
   - no

7. Security Risk
   - yes
   - no

8. Confidence Score
   - numeric value between 0 and 1

Categorical features should be encoded using one-hot encoding where appropriate.

Reasoning features are stored separately and merged using commit_id.

Important:

CodeBERT embeddings are already generated and PCA-reduced.

Do not generate code for:
- CodeBERT embedding extraction
- Transformer inference
- PCA fitting

Experiments should directly load precomputed PCA-reduced embeddings.

LLM reasoning features are generated externally and merged using commit_id.

Experiments should treat reasoning features as structured numerical features.
